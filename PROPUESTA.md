## Idea del proyecto

BarberFlow es una aplicación web orientada a la gestión de reservas para barberías. El sistema permitirá administrar clientes, servicios, barberos,
horarios y reservas desde una plataforma accesible desde el navegador.

La aplicación busca mejorar la organización de la barbería, reducir tiempos de espera y facilitar 
la gestión de citas tanto para los clientes como para los administradores del negocio.

---

## Usuario objetivo

- Barberías pequeñas y medianas
- Barberos independientes
- Clientes que desean reservar citas de manera rápida y organizada

---

## Problema que resuelve

Actualmente muchas barberías gestionan las citas manualmente mediante llamadas, mensajes o listas físicas, 
lo que genera desorganización, pérdida de turnos y largas esperas.

BarberFlow permitirá centralizar la administración de reservas, horarios y servicios en una sola plataforma web, 
mejorando la atención al cliente y la organización del negocio.


## Páginas principales que imaginamos

Página de inicio: 
Un "Landing Page" que presente la barbería, los servicios destacados  y un botón claro para "Agendar Cita".  

Página de Reservas (Dashboard de Cliente): 
Un calendario o formulario donde el cliente elija el barbero, el servicio y el horario disponible.  

Panel de Administración:
Una vista principal para el dueño con el resumen de las citas del día.  

Formulario de Registro/Edición: 
Páginas para dar de alta nuevos clientes , servicios (con sus precios) y barberos.  

Listado de Gestión: 
Tablas interactivas para visualizar y administrar la base de datos de clientes, empleados y el historial de reservas. 
(Opcional) Pantalla de Cola de Atención: 
Si decides implementar la mejora avanzada, una página que muestre el turno actual y los clientes en espera. 

## Tecnologías propuestas
Frontend: HTML5, CSS, Bootstrap (obligatorios según tu guía) y JavaScript para la interactividad del lado del cliente. 
Backend: Node.js (es muy común usarlo junto con JavaScript del frontend). 
Base de datos: MySQL (ideal para manejar las relaciones entre tablas como clientes, servicios y reservas)
